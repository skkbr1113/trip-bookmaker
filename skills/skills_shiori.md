# skills/shiori.md

## 目的
旅行メンバー全員が旅行前・当日に参照できるしおりページ（index.html）を作ること。
brand.mdとproject.mdの内容を踏まえて実装すること。

## セクション構成
以下の6セクションを固定構成とすること。

1. ヒーロー（タイトル・日程・旅行先）
2. メンバー（アバター・名前・係）
3. 1日目スケジュール（タイムライン形式）
4. 2日目スケジュール（後述のルールに従うこと）
5. 設備・アメニティ（宿泊先情報から自動調査）
6. 持ち物リスト（設備情報をもとに自動生成）

## 変数（毎回質問すること）
しおりページ生成前に以下を必ず確認すること。

- 旅行先・都道府県
- 旅行日程（〇月〇日〜〇泊）
- 宿泊先（名前・URL）
- メンバーの名前と担当係（人数分）
- スポット候補（名前・カテゴリ・URL）
- 1日目のタイムライン（時刻・内容）
- 2日目のスケジュールは決まっているか
- アバター画像（外部ツールで用意後に渡してもらう）
- デザインマップ画像（外部ツールで用意後に渡してもらう）
- スマホ特化（max-width 420px）にするか、PC対応も含めるか

## ヒーローセクションの仕様
- 背景色：#085041（ダークグリーン）
- テキスト：中央揃え・白文字
- 表示内容：タイトル・日程のみ（キャッチコピーは不要）

## メンバーセクションの仕様
- アバター画像は256px正方形・顔が中央に来るようにクロップして円形表示（border-radius: 50%）にすること。
- 名前・担当係をアバターの下に表示すること。
- アバター画像はbase64でHTMLに埋め込むこと。
- アバター画像が未渡しの場合は「検討中」と表示すること。

## 1日目スケジュールの実装ルール
- タイムライン形式で時刻・内容を縦に並べること。
- 各項目の内容に応じて近いアイコン（絵文字）を自動で選んで付けること。
- アイコン選定の参考（内容に合わせて柔軟に判断すること）：
  - 移動・出発：🚗 🚃 🚌 ✈️
  - 食事：🍽️ 🍜 🍣 ☕
  - 観光：🗺️ 📸 🏯
  - 温泉・入浴：♨️ 🛁
  - 宿泊・チェックイン：🏡 🛏️
  - 買い物：🛍️ 🛒
  - レジャー・アクティビティ：🎿 🏄 🎣

## 2日目スケジュールの実装ルール
生成前に「2日目のスケジュールは決まっていますか？」と確認すること。

- **決まっている場合**：1日目と同じタイムライン形式で実装すること。
- **決まっていない場合**：デザインマップにスポット候補のピンを複数配置し、「宿でみんなで決めよう」というメッセージを添えて表示すること。
- **スポット候補が0件の場合**：マップセクションは表示すること。中央に「？」を円形で表示し、スポットが未定であることを示すこと。

## 2日目マップの実装ルール（スケジュール未定の場合）
- イラストマップ画像はユーザーが外部ツールで用意して渡すこと。
- マップ画像が未渡しの場合は「検討中」と表示すること。
- ピンの座標はマップ画像を見せてもらいClaudeCodeが自動で配置し、その後ラリーしながら微調整すること。
- スポットのピンはパーセント指定（left・top）で配置すること。
- ピンのラベルは1行・短いテキストにすること（テキスト溢れ防止）。

## ピン詳細パネルの仕様
タップ時に表示するパネルには以下の情報を含めること。

- スポット名
- カテゴリ（温泉・観光・食べ歩きなど）
- 滞在目安時間（例：約1〜2時間）
- 一言説明
- GoogleマップリンクボタンとWebサイトリンクボタン

## セクション区切りの仕様
セクション間の区切りはボタニカルSVG（葉・草花モチーフ・緑系）を使うこと。

## ナビゲーションバーの仕様
ページ下部に固定ナビを設置すること。

```html
<nav class="bottom-nav">
  <a href="index.html" class="nav-item active">📋 しおり</a>
  <a href="warikan.html" class="nav-item">💰 割り勘</a>
</nav>
```

## フォントサイズ・余白の基準
前回の実装値を踏襲すること。index.htmlが存在する場合は必ず参照して揃えること。

## 設備・アメニティの調査ルール
宿泊先のURLをもとにWebを検索・調査し、以下の情報を取得すること。

- 設備一覧（キッチン・調理器具・バス・トイレ・洗濯機など）
- アメニティ一覧（歯ブラシ・タオル・シャンプー・パジャマなど）
- 宿泊先が提供しないもの（持参が必要なもの）を特定すること。

調査できない項目は「要確認」として明示すること。

## 持ち物リストの生成ルール
設備・アメニティの調査結果をもとに持ち物リストを自動生成すること。

- 宿泊先にないアメニティ → 持参必須として追加すること。
- キッチンあり → 食材・調理器具を検討リストに追加すること。
- 洗濯機なし → 着替えを多めに持参する旨を追加すること。

生成したリストはHTML化前に必ず確認を取ること。
「この内容で進めてよいですか。追加・削除があれば教えてください。」と聞くこと。
確認が取れてからHTMLに組み込むこと。

## 技術仕様
- 単一HTMLファイル（HTML・CSS・JS内包）にすること。
- 外部依存はGoogle Fonts（Klee One）のみにすること。
- 画像はbase64でHTMLに埋め込み、自己完結ファイルにすること。
- スマホ特化かどうかは変数確認の結果に従うこと。
