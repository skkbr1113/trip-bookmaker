# trip-bookmaker 🏔️

友人グループの旅行ごとに「**しおり**」と「**割り勘**」の2ページをつくる旅行サイトジェネレーター。
Claude Code に「旅行行く」と話しかけて質問に答えるだけで、スマホ特化の旅行サイトが出来上がり、GitHub Pages で公開されます。

📖 公開中のしおり: https://skkbr1113.github.io/trip-bookmaker/20260711-12_karuizawa/

## 使った技術

| 領域 | 技術 |
|---|---|
| フロントエンド | HTML / CSS / JavaScript（フレームワークなし・1ページ1ファイル完結） |
| リアルタイム同期 | Firebase Realtime Database（compat SDK・CDN読み込み） |
| ホスティング | GitHub Pages |
| フォント | Klee One（Google Fonts） |
| 開発エージェント | Claude Code（CLAUDE.md 司令塔 + スキル構成） |

## 何と連携しているか

- **Firebase Realtime Database** — 割り勘の支払いデータを全員のスマホにリアルタイム同期。ログイン不要でURLを開くだけ
- **Google マップ** — しおりのスポット詳細から各スポットの地図へワンタップ遷移
- **GitHub Pages** — push するだけで公開。旅行メンバーにはURLを共有するだけ

## 工夫しているポイント

### サイトのデザイン・体験
- **スマホ完結・自己完結ファイル**: 最大幅420px、アバター画像はbase64埋め込みで index.html 単体でも動く
- **動くしおり**: 浅間山の噴煙・星・蛍・流れ星が動くSVGヒーロー、出発日までのカウントダウン、スクロールに合わせて灯るタイムライン、旅行当日は現在時刻に「📍イマココ」バッジ
- **持ち物チェックリスト**: タップでチェック（localStorageに保存）、🚗が🏔️へ進む準備度メーター、100%で紙吹雪
- **遊び心**: 夜のBBQ項目をタップすると花火が上がる隠し演出
- **割り勘の精算アルゴリズム**: 最小送金回数になるよう貪欲法で相殺。端数は支払者負担で1円単位まで合う
- **メンバー別収支の可視化**: 誰がいくら払って「もらう / はらう / ぴったり」かをアバターとバーで表示
- **アクセシビリティ**: prefers-reduced-motion 対応（アニメーション苦手な人には静的表示）

### 開発フローの自動化（Claude Code）
- **CLAUDE.md が司令塔**: 起動時に project.md（目的）→ brand.md（世界観）→ skills/（ページ仕様）を読む構成
- **trip-new スキル**: 「旅行行く」で起動。行き先・日程・メンバーを会話形式で質問し、前回の旅行のデザインを引き継いで新しいしおりを生成
- **publish スキル**: 「公開して」で起動。コミット前にセルフレビュー→修正を自動で繰り返し、コミットメッセージ候補を提示してから push

## 運用メモ

- Firebase Realtime Database はテストモード（誰でも読み書き可）で短期運用。**旅行が終わったらルールを締めるかデータを削除する**
- Firebase の apiKey はウェブ公開前提のキーのためソース直書きでOK
- 新しい旅行では割り勘データを `trips/<フォルダ名>/payments` に分離し、過去の旅行と混ざらないようにする
